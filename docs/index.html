Overview

SonarVision is a production-grade framework for real-time underwater sonar processing. It combines CUDA-accelerated beamforming with constraint-theory-based physics modeling to deliver deterministic, low-latency sonar data interpretation on edge hardware (Jetson, FPGA).

Capabilities

CUDA Beamforming Pipeline

Multi-kernel sonar processing with temporal smoothing and adaptive peak detection. CUDA Graph-accelerated for minimum latency.

CUDA C

Marine Physics Engine

Francois-Garrison absorption, Rayleigh scattering, Mackenzie sound speed, thermocline profiling, Jerlov water typing, Snell refraction — all deterministic and bit-identical across instances.

Python C

MEP Protocol

16-byte packed Marine Engineering Protocol frames for low-latency UDP transport of sonar depth profiles, confidence metrics, and physics snapshots.

C Rust

SonarVision CLI

Command-line tool for pinging, scanning, and streaming sonar data. Includes live depth profiling, physics querying, waterfall visualization.

Python Rust

Dashboard Widget

Live WebSocket-connected sonar panel with waterfall canvas, depth profile bar chart, and real-time metrics (FPS, latency, confidence).

Docker TypeScript

TypeScript Agent Tool

open-agents-compatible tool for querying sonar vision data. Three actions: infer (depth → frame), physics (deterministic params), health (server status).

Quick Start

Install

# Python pip install sonar-vision # Docker docker pull superinstance/sonar-vision docker run -p 8080:8080 superinstance/sonar-vision

Basic Usage

from sonar_vision import MarinePhysicsScope # Compute physics at 15m depth scope = MarinePhysicsScope(depth_m=15.0).compute() print(f"Water: {scope.water_type_name}") print(f"Visibility: {scope.visibility}m") print(f"Sound speed: {scope.sound_speed} m/s") # → Water: Oceanic Type II # → Visibility: 6.76m # → Sound speed: 1527 m/s

Ping Sonar

sonar-vision ping --depth 30 --frequency 200kHz # → Absorption: 0.241 m^-1 # → Scattering: 0.0096 m^-1 # → Visibility: 6.78m # → Seabed: 0.430 (reflection coefficient)

Deployment

Docker

Single-container deployment with Prometheus monitoring endpoint. Exposes port 8080 for REST API and 8081 for metrics.

docker-compose up # → SonarVision running on :8080

Jetson Edge

Native CUDA deployment on Jetson Orin/AGX. Cross-compiled with CUDA 12.x and optimized for 8W-15W power envelope.

Observability

Prometheus metrics for ping latency, beamforming throughput, physics op latency, and memory utilization. Grafana dashboard included.

API Reference

# REST Endpoints POST /api/v1/ping # Sonar ping with physics POST /api/v1/beamform # Beamform raw sensor data GET /api/v1/health # Server health + metrics GET /api/v1/physics # Physics model query # WebSocket WS /ws/stream # Live sonar data stream

Example Request

curl -X POST http://localhost:8080/api/v1/ping \ -H "Content-Type: application/json" \ -d '{"depth": 15, "frequency_khz": 200}' # Response: # { # "depth": 15, # "water_type": "Oceanic Type II", # "absorption": 0.241, # "scattering": 0.0102, # "attenuation": 0.252, # "visibility": 6.76, # "sound_speed": 1526.9, # "temperature": 22.0, # "seabed_reflectivity": 0.464 # }
